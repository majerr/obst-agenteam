# OBST project outline

## Problem Statement

Teams in the Advanced Analytics and AI Division (AAAI) are struggling
to deliver high-value products. Building the products is not the
issue - one team delivered a prototype multi-layer, working,
end-to-end prototype in a fortnight - but teams of data scientists are
spending weeks discussing user value, user research, infrastructure,
governance, devops and delivery, none of which are core data science
skills.

## Suggested solution(s)

A well designed framework could accelerate the process between having
an idea and being ready to write code.

### OBST

OBST (Outlines, Briefs, Specs, Tests) is an idea for an AI-first
delivery framework to accelerate teams' progress from idea to
implementation readiness. Benefits include:

* Using AI to accelerate development by 
  - challenging ideas, 
  - reviewing suggestions, 
  - generating artefacts, 
  - managing issues
* The process emits it's own audit trail
* The process results in a repository that is aligned on intention
  with consistent artefacts (e.g. decision records, specs, tests),
  providing
* OBST provides a narrative structure for development that makes it
  easier to quickly understand project progress
* The process focusses on explicit decision making and comprehensive
  record keeping to facilitate project governance.

## Aims

The aim of this project is to define and develop a framework and
supporting materials (definitions, tutorial, examples) for use in
AAAI. 

## Context

The immediate context for the use of this framework is a team of UK
Civil Servants. Any use of AI or other technology must be security
assured and procurable.
